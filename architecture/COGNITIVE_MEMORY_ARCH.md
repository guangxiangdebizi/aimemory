# 类人记忆架构设计

## 设计哲学

不是给 LLM 加一个"记忆功能"，而是从人类认知架构出发，
设计一套记忆系统，使其写入、保留、遗忘、回忆的行为模式尽可能接近人脑。

## 总体架构

```
用户输入
  │
  ▼
┌─────────────────────────────┐
│  Layer 1: 注意力过滤         │  ← 决定什么值得记
│  (Selective Encoding)        │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Layer 2: 工作记忆           │  ← 当前对话上下文
│  (Working Memory)            │
│  = LLM Context Window        │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Layer 3: 记忆巩固           │  ← 短期 → 长期的转化
│  (Memory Consolidation)      │
│  - 重要性评估                │
│  - 情感标记                  │
│  - 关联编码                  │
│  - 压缩与结构化              │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Layer 4: 长期记忆库          │
│  (Long-term Memory Store)    │
│                              │
│  ┌─────────┐ ┌──────────┐   │
│  │ 语义记忆 │ │ 情景记忆  │   │
│  │ Semantic │ │ Episodic  │   │
│  └─────────┘ └──────────┘   │
│  ┌─────────┐ ┌──────────┐   │
│  │ 程序记忆 │ │ 用户画像  │   │
│  │Procedural│ │ Profile   │   │
│  └─────────┘ └──────────┘   │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Layer 5: 遗忘机制           │  ← 周期性运行
│  (Forgetting Engine)         │
│  - 时间衰减                  │
│  - 访问频率衰减              │
│  - 冲突消解                  │
│  - 碎片合并压缩              │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Layer 6: 回忆重建           │  ← 下次提问时触发
│  (Adaptive Recall)           │
│  - 多路候选召回              │
│  - 动态排序                  │
│  - 边界扩展                  │
│  - 记忆融合                  │
│  - 上下文预算分配            │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Layer 7: 主模型推理          │
│  (Main LLM Inference)        │
│  接收: 当前输入 + 重建记忆    │
│  输出: 回答                  │
└─────────────────────────────┘
```

## 各层详细设计

### Layer 1: 注意力过滤（Selective Encoding）

**对应人类机制：** 感觉记忆 → 短期记忆的门控

**职责：** 判断当前对话中哪些信息值得写入长期记忆

**输入：** 当前轮对话内容（用户输入 + 模型回复）

**输出：** 标记为"值得记忆"的信息片段 + 重要性分数

**设计要点：**
- 不是所有对话都值得记——闲聊、确认、重复不需要存
- 需要识别：新偏好、新事实、重要决策、情感表达、明确指令
- 可以用小模型做分类，也可以用规则 + LLM 判断

**参考论文：** 认知科学综述（选择性编码）

---

### Layer 2: 工作记忆（Working Memory）

**对应人类机制：** 短期/工作记忆

**职责：** 维护当前对话的即时上下文

**实现：** 就是 LLM 的 context window，现有架构天然具备

**设计要点：**
- 容量有限（对应人类 7±2 限制）
- 对话结束后清空（但重要内容已被 Layer 1 筛出）
- 可以考虑 MemGPT 式的分页管理来扩展有效容量

---

### Layer 3: 记忆巩固（Memory Consolidation）

**对应人类机制：** 记忆巩固（睡眠整理、复述强化）

**职责：** 将 Layer 1 筛出的信息转化为结构化长期记忆

**触发时机：**
- 对话结束后（类似"睡眠整理"）
- 周期性批处理（每 N 轮对话整理一次）
- 检测到重要事件时实时触发

**处理流程：**
1. 提取关键信息（实体、关系、事件、偏好）
2. 评估重要性（影响范围、情感强度、新颖度）
3. 和已有记忆做关联（是更新还是新增？有没有冲突？）
4. 压缩为结构化格式存入 Layer 4

**参考论文：** EM-LLM（事件分割）、GSW（结构化表示）

---

### Layer 4: 长期记忆库（Long-term Memory Store）

**对应人类机制：** 长期记忆（语义 + 情景 + 程序）

**四类记忆：**

#### 4a. 语义记忆（Semantic Memory）
抽象化的长期知识，不带具体时间。

```json
{
  "type": "semantic",
  "content": "用户偏好极简设计风格",
  "confidence": 0.92,
  "source_count": 5,
  "last_reinforced": "2026-04-01"
}
```

#### 4b. 情景记忆（Episodic Memory）
具体事件，带时间、情境、因果。

```json
{
  "type": "episodic",
  "content": "用户在讨论 UI 配色时明确表示不喜欢蓝色",
  "timestamp": "2026-03-15T14:30:00",
  "context": "UI 设计讨论",
  "emotion": "frustration",
  "related_entities": ["UI", "配色", "蓝色"]
}
```

#### 4c. 程序记忆（Procedural Memory）
用户习惯的交互模式和工作流程。

```json
{
  "type": "procedural",
  "content": "用户习惯先给需求概述，再逐步细化",
  "frequency": "high",
  "last_observed": "2026-04-05"
}
```

#### 4d. 用户画像（Profile Memory）
长期稳定的身份和偏好信息。

```json
{
  "type": "profile",
  "content": "用户是后端开发者，主要用 Python",
  "stability": "high",
  "first_observed": "2026-01-10"
}
```

**存储方案选项：**
- 向量数据库（语义检索）
- 图数据库（关系和因果）
- 结构化 JSON/SQL（精确查询）
- 混合方案（推荐）

**参考论文：** SYNAPSE（图结构）、MemoryBank（衰减管理）

---

### Layer 5: 遗忘机制（Forgetting Engine）

**对应人类机制：** 艾宾浩斯遗忘、干扰遗忘、动机性遗忘

**职责：** 周期性清理、压缩、合并记忆

**衰减公式（参考 FadeMem）：**

```
strength(t) = initial_strength × exp(-λ × t)

其中 λ = base_rate / (semantic_relevance × access_frequency)
```

- 语义相关性高 → λ 小 → 衰减慢
- 访问频率高 → λ 小 → 衰减慢
- 时间越久 → strength 越低

**遗忘策略：**

| 操作 | 触发条件 | 效果 |
|------|---------|------|
| 自然衰减 | 周期性 | strength 降低 |
| 碎片合并 | 多条相关记忆 strength 都低 | 合并为一条摘要 |
| 冲突覆盖 | 新记忆和旧记忆矛盾 | 旧记忆标记为过时 |
| 彻底删除 | strength 低于阈值且无关联 | 从存储中移除 |
| 强化恢复 | 被重新访问/提及 | strength 回升 |

**参考论文：** MemoryBank（艾宾浩斯）、FadeMem（多因素衰减）

---

### Layer 6: 回忆重建（Adaptive Recall）

**对应人类机制：** 回忆是重建过程，不是精确读取

**职责：** 根据当前 query 动态组装最相关的记忆上下文

**这是和传统 RAG 差异最大的一层。**

#### 步骤 1: 多路候选召回
同时从多个通道召回候选记忆：
- 语义相似度召回
- 时间邻近召回
- 图关联召回（扩散激活）
- Profile 强制召回
- 高优先级规则召回

#### 步骤 2: 动态排序
不是固定 topk，而是根据 query 动态决定：
- 每条候选的综合得分 = 相关性 + 重要性 + 新鲜度 + 用户权重
- 得分阈值动态调整（简单问题少取，复杂问题多取）

#### 步骤 3: 边界扩展
命中一个记忆点后，动态决定是否扩展上下文：
- 只取当前条目
- 扩展到同一事件
- 扩展到同一主题段
- 扩展到相关事件链

#### 步骤 4: 记忆融合
多条相关记忆可能需要合并为一个连贯叙述，而不是碎片罗列。

#### 步骤 5: 上下文预算分配
根据总 token 预算，决定每类记忆分配多少空间：

```
总预算 = N tokens
├── Profile 记忆: 固定分配 ~10%
├── 高优先级情景记忆: ~30%
├── 语义记忆摘要: ~20%
├── 当前任务相关历史: ~30%
└── 缓冲区: ~10%
```

**参考论文：** SYNAPSE（扩散激活）、EM-LLM（两阶段检索）、GSW（结构化表示）

---

### Layer 7: 主模型推理（Main LLM Inference）

**职责：** 接收重建后的记忆 + 当前输入，生成回答

**输入格式：**
```
[System Prompt]
[Profile Memory Block]
[Relevant Episodic Memories]
[Semantic Context]
[Current Conversation]
[User Query]
```

**可选增强：**
- Adapter 微调：让模型更擅长利用结构化记忆
- EpMAN 式注意力：在 attention 层面融合记忆（而非纯文本拼接）
- MEMORYLLM 式内生记忆：核心记忆写入 latent space

## 与现有论文的对应关系

| 架构层 | 对应论文 |
|--------|---------|
| Layer 1 注意力过滤 | 认知科学综述（选择性编码） |
| Layer 2 工作记忆 | MemGPT（虚拟上下文管理） |
| Layer 3 记忆巩固 | EM-LLM（事件分割）、GSW（结构化） |
| Layer 4 长期记忆库 | SYNAPSE（图结构）、MemoryBank |
| Layer 5 遗忘机制 | MemoryBank（艾宾浩斯）、FadeMem（多因素） |
| Layer 6 回忆重建 | SYNAPSE（扩散激活）、EM-LLM（两阶段） |
| Layer 7 主模型 | MEMORYLLM / M+（内生）、EpMAN（注意力） |
