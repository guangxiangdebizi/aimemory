# MemoryBank: Enhancing Large Language Models with Long-Term Memory

## 基本信息

| 项目 | 内容 |

| **Venue** | AAAI 2024 |
| **作者** | Wanjun Zhong, Lianghong Guo, Qiqi Gao, He Ye, Yanlin Wang |

## 为什么这篇重要

**第一篇在顶会上将艾宾浩斯遗忘曲线引入 LLM 记忆系统的工作。**

它不只是"存记忆"，还做了"忘记忆"——根据时间和重要性让记忆强度自然衰减。
这是最早认真对待"AI 也需要遗忘"这件事的工作之一。

## 核心架构（原文技术细节）

### 整体流程

```
用户输入 → 记忆检索 → 相关记忆注入上下文 → LLM 生成回复
                                                    ↓
                                        对话内容写入记忆库
                                                    ↓
                                        记忆库执行遗忘更新
```

### 记忆更新：艾宾浩斯遗忘曲线

每条记忆有一个 **强度值（strength）**，随时间按遗忘曲线衰减：

```
R(t) = e^{-t/S}

其中：
  R(t) = 记忆保留率
  t = 距上次访问的时间间隔
  S = 记忆强度（由重要性决定）
```

- 时间越久 → R(t) 越低 → 记忆越弱
- 重要性越高 → S 越大 → 衰减越慢
- 被重新访问 → t 重置为 0 → 强度恢复（类似人类复习效果）

### 记忆存储结构

每条记忆包含：
- **内容**：对话摘要或关键信息
- **时间戳**：写入时间
- **强度值**：当前记忆强度
- **重要性标签**：由 LLM 评估

### 三大能力

1. **记忆召回**：根据当前对话语义检索相关历史记忆
2. **持续更新**：新对话自动写入，旧记忆按遗忘曲线自动衰减
3. **人格理解**：通过长期交互积累用户画像，理解性格和偏好

### 落地应用：SiliconFriend

基于 MemoryBank 的长期陪伴型聊天机器人：
- 用心理对话数据微调
- 能提供共情回应
- 能回忆相关历史
- 能理解用户人格

## 关键结果

- 兼容闭源（ChatGPT）和开源（ChatGLM）模型
- 在长期对话场景中显著优于无记忆基线
- 定性分析（真实用户对话）+ 定量分析（ChatGPT 模拟多种人格的长期对话）双重验证
- 模拟实验中 ChatGPT 扮演不同性格用户，覆盖多种话题，验证记忆的长期有效性

## 与本项目的关系

**直接相关度：⭐⭐⭐⭐⭐**

这篇直接对应我们的"遗忘机制"需求。
艾宾浩斯遗忘曲线是最经典的认知科学记忆模型之一，这篇把它工程化了。

后续 FadeMem 在此基础上做了更精细的多因素自适应衰减。

## 与 MEMORYLLM 的关键区别

| 维度 | MemoryBank | MEMORYLLM |
|------|-----------|-----------|
| 记忆位置 | 外部存储 | 模型 latent space 内部 |
| 遗忘机制 | 显式艾宾浩斯曲线 | 隐式随机丢弃（也是指数衰减） |
| 是否需要训练 | 不需要 | 需要专门训练 |
| 写入精度 | 文本级（精确） | latent 级（模糊） |
| 可解释性 | 高（能看到每条记忆） | 低（记忆溶解在参数里） |

有趣的是，MEMORYLLM 论文明确引用了 MemoryBank，
并承认其随机丢弃机制的灵感来自艾宾浩斯遗忘曲线。
