## 五个方向，一句话版

| 方向 | 一句话理解 | 难点 |
|------|------------|------|
| A. RL 驱动的记忆策略 | 让模型自己学会"什么时候记、什么时候删、什么时候更新" | 训练重，竞争激烈 |
| B. 记忆巩固 | 让短期记忆在空闲时被整理、合并、升级为长期记忆 | 目前最缺系统化方法 |
| C. 因果驱动检索 | 不只找"像"的记忆，而是找"真正导致当前问题的关键记忆" | 因果关系难标注 |
| D. 动态事件分割 + 结构化表示 | 先把经历切成合理事件，再把每个事件存成结构化记忆 | 需要多模块配合 |
| E. 多模态 / 具身记忆 | 让 agent 记住看过什么、走过哪里、做过什么 | 工程量最大 |

## 当前推荐优先级

基于现有文献和可落地性，当前更值得优先考虑的是：

1. `B. 记忆巩固`
2. `C. 因果驱动检索`
3. `D. 动态事件分割 + 结构化表示`
4. `A. RL 驱动的记忆策略`
5. `E. 多模态 / 具身记忆`

## 这些文档怎么读

每份文档都尽量回答 8 个问题：

1. 这个方向到底在做什么
2. 它为什么重要
3. 它和现有论文是什么关系
4. 一个新手怎么把它想象成系统模块
5. 现在卡在哪里
6. 真做研究时可以怎么做实验
7. 它的风险是什么
8. 它更适合硕士课题、论文选题，还是长期系统工程

如果你后面要继续收敛选题，最实用的路径是：

1. 先读 `B`
2. 再读 `C`
3. 如果想要更工程、更组合创新，再读 `D`
